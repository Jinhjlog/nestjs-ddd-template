# 관리자 인증 (Admin Auth)

## 1. 배경 및 문제 정의

백오피스 시스템에는 운영자가 콘텐츠와 회원 데이터를 관리하기 위한 전용 관리자 인증 체계가 필요하다. 일반 사용자 인증과는 별개로 관리자 전용 JWT를 발급하고, 역할(SUPER_ADMIN / ADMIN)에 따라 접근 가능한 기능을 제한해야 한다. 리프레시 토큰을 DB에 해시 형태로 저장하여 토큰 탈취 시 재사용을 방지한다.

## 2. 목표

- 관리자 전용 로그인/로그아웃/토큰 갱신 API를 제공한다.
- 액세스 토큰(단기) + 리프레시 토큰(7일)의 이중 토큰 방식을 적용한다.
- 리프레시 토큰 Rotation을 적용해 탈취된 토큰의 재사용을 원천 차단한다.
- 역할 기반 접근 제어(RBAC)를 통해 SUPER_ADMIN만 관리자 계정 관리를 수행할 수 있도록 한다.
- 만료된 리프레시 토큰은 자동 정리 스케줄러를 통해 DB에서 제거한다.

## 3. 사용자 시나리오

### 시나리오 1. 관리자 로그인

1. 관리자가 로그인 화면에서 아이디(`loginId`)와 비밀번호(`password`)를 입력한다.
2. 서버는 아이디로 관리자를 조회하고 비밀번호를 bcrypt로 비교한다.
3. 아이디가 존재하지 않거나 비밀번호가 일치하지 않으면, 계정 존재 여부를 노출하지 않기 위해 동일한 오류 메시지(`INVALID_CREDENTIALS`)를 반환한다.
4. 계정이 비활성화(`isActive = false`) 상태이면 403 오류(`ADMIN_ACCOUNT_INACTIVE`)를 반환한다.
5. 인증 성공 시 액세스 토큰과 리프레시 토큰을 발급하고, 마지막 로그인 시각(`lastLoginAt`)을 갱신한다.

### 시나리오 2. 토큰 갱신 (Token Rotation)

1. 클라이언트가 액세스 토큰 만료 후 리프레시 토큰을 본문에 담아 `/admin-auth/refresh`를 호출한다.
2. 서버는 토큰 형식(`tokenId:rawToken`)을 파싱하고, DB에서 `tokenId`로 리프레시 토큰 레코드를 조회한다.
3. 만료 여부를 확인하고, bcrypt로 원본 토큰과 해시를 비교하여 유효성을 검증한다.
4. 검증 성공 시 기존 리프레시 토큰을 삭제(일회용 rotation)하고, 새로운 토큰 쌍을 발급한다.
5. 검증 실패 시 각 단계에 맞는 401 에러를 반환한다.

### 시나리오 3. 관리자 로그아웃

1. 클라이언트가 리프레시 토큰을 본문에 담아 `/admin-auth/logout`을 호출한다.
2. 서버는 리프레시 토큰에서 `tokenId`를 파싱하여 DB에서 해당 레코드를 삭제한다.
3. 204 No Content를 반환한다.

### 시나리오 4. 인증이 필요한 API 접근

1. 클라이언트가 `Authorization: Bearer <accessToken>` 헤더와 함께 보호된 API를 호출한다.
2. `AdminJwtGuard`가 헤더에서 Bearer 토큰을 추출하고 서명을 검증한다.
3. 토큰 페이로드에서 `adminId`를 추출하여 DB에서 관리자를 조회하고 활성 상태를 확인한다.
4. 검증 성공 시 요청 객체에 `AuthenticatedAdmin(adminId, role, name)`을 주입한다.

### 시나리오 5. 최고 관리자 전용 기능 접근

1. SUPER_ADMIN 관리자가 보호된 엔드포인트를 호출한다.
2. `AdminJwtGuard`가 JWT를 검증하고, `AdminRoleGuard`가 `role`이 `SUPER_ADMIN`인지 확인한다.
3. `ADMIN` 역할을 가진 관리자가 접근하면 403 오류를 반환한다.

## 4. 기능 요구사항

### 인증

- [x] 관리자 아이디 + 비밀번호로 로그인
- [x] 로그인 성공 시 액세스 토큰 + 리프레시 토큰 발급
- [x] 로그인 성공 시 `lastLoginAt` 갱신
- [x] 아이디 미존재 / 비밀번호 불일치 시 동일한 오류 반환
- [x] 비활성화 계정 로그인 차단 (403)
- [x] 리프레시 토큰으로 토큰 갱신 (Token Rotation)
- [x] 리프레시 토큰으로 로그아웃

### 토큰 관리

- [x] 리프레시 토큰 bcrypt 해시 DB 저장
- [x] 리프레시 토큰 형식: `{tokenId}:{rawToken}`
- [x] 리프레시 토큰 유효 기간: 7일
- [x] 만료된 리프레시 토큰 자동 삭제 스케줄러

### 접근 제어

- [x] `AdminJwtGuard`: Bearer JWT 검증 + DB 관리자 활성 상태 확인
- [x] `AdminRoleGuard`: 메타데이터 기반 역할 검증
- [x] `@AdminAuth()` 데코레이터: JWT 인증 가드 + Swagger 문서화
- [x] `@RequireSuperAdmin()` 데코레이터: JWT 인증 + SUPER_ADMIN 역할 검증
- [x] `@CurrentAdmin()` 파라미터 데코레이터: 인증된 관리자 정보 주입

### 관리자 계정 관리 (SUPER_ADMIN 전용)

- [x] 관리자 등록 (`loginId`, `password`, `name` 필수, `role`은 항상 `ADMIN`)
- [x] `loginId` 중복 검증 → 409
- [x] 전체 관리자 목록 조회 (소프트 딜리트 제외)
- [x] 관리자 상세 조회

### 비밀번호 검증

- [x] 최소 8자, 최대 25자
- [x] 특수문자 1개 이상 포함 필수

## 5. 범위

### 포함

- 관리자 로그인 / 토큰 갱신 / 로그아웃 API
- JWT 기반 관리자 인증 가드 + 역할 기반 접근 제어
- 리프레시 토큰 DB 저장 및 해시 검증
- 만료된 리프레시 토큰 자동 정리 스케줄러
- SUPER_ADMIN 전용 관리자 계정 등록 / 조회 API

### 미포함

- 관리자 계정 삭제(소프트 딜리트) API
- 비밀번호 변경 / 초기화 API
- 로그인 실패 횟수 제한 및 계정 잠금
- OAuth / SSO 연동
