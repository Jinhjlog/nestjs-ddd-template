# 사용자 인증 (User Auth)

## 1. 배경 및 문제 정의

서비스 사용자를 위한 이메일/비밀번호 기반 인증 체계가 필요하다. 관리자 인증과는 별개의 JWT 컨텍스트로 분리하여 토큰 간 호환을 차단하고, 리프레시 토큰 Rotation으로 보안을 강화한다.

## 2. 목표

- 이메일/비밀번호 기반 회원가입 및 로그인 API를 제공한다.
- 액세스 토큰(단기) + 리프레시 토큰(7일)의 이중 토큰 방식을 적용한다.
- 리프레시 토큰 Rotation을 적용해 탈취된 토큰의 재사용을 차단한다.
- 사용자 프로필 조회 기능을 제공한다.
- 관리자가 사용자 목록/상세를 조회하고 활성 상태를 관리할 수 있도록 한다.

## 3. 사용자 시나리오

### 시나리오 1. 회원가입

1. 사용자가 이메일, 비밀번호, 이름(선택), 전화번호(선택)를 입력하여 회원가입을 요청한다.
2. 서버는 이메일 형식과 비밀번호 규칙(8자 이상, 특수문자 포함)을 검증한다.
3. 이미 가입된 이메일이면 409 오류(`EMAIL_ALREADY_EXISTS`)를 반환한다.
4. 검증 통과 시 비밀번호를 bcrypt로 해시하여 사용자를 생성하고, 토큰 쌍을 발급한다.

### 시나리오 2. 로그인

1. 사용자가 이메일과 비밀번호를 입력하여 로그인을 요청한다.
2. 서버는 이메일로 사용자를 조회하고 비밀번호를 bcrypt로 비교한다.
3. 이메일 미존재 또는 비밀번호 불일치 시 동일한 오류(`INVALID_CREDENTIALS`)를 반환한다.
4. 계정이 비활성화 상태이면 403 오류(`USER_ACCOUNT_INACTIVE`)를 반환한다.
5. 인증 성공 시 액세스 토큰과 리프레시 토큰을 발급한다.

### 시나리오 3. 토큰 갱신 (Token Rotation)

1. 클라이언트가 리프레시 토큰을 본문에 담아 `/user-auth/refresh`를 호출한다.
2. 서버는 토큰 형식(`tokenId:rawToken`)을 파싱하고 DB에서 조회, 만료/해시 검증을 수행한다.
3. 검증 성공 시 기존 토큰을 삭제하고 새 토큰 쌍을 발급한다.

### 시나리오 4. 로그아웃

1. 클라이언트가 리프레시 토큰을 본문에 담아 `/user-auth/logout`을 호출한다.
2. 서버는 DB에서 해당 리프레시 토큰을 삭제하고 204를 반환한다.

### 시나리오 5. 프로필 조회

1. 인증된 사용자가 `/users/me`를 호출한다.
2. `UserJwtGuard`가 JWT를 검증하고 사용자 정보를 주입한다.
3. 사용자 프로필(이메일, 이름, 전화번호 등)을 반환한다.

### 시나리오 6. 관리자의 사용자 관리

1. SUPER_ADMIN 또는 ADMIN 관리자가 사용자 목록을 조회한다.
2. 이름, 활성 상태로 필터링하고 페이지네이션을 적용한다.
3. 특정 사용자의 상세 정보를 조회하거나 활성/비활성 상태를 변경한다.

## 4. 기능 요구사항

### 인증

- [x] 이메일 + 비밀번호로 회원가입
- [x] 이메일 중복 검증 → 409(`EMAIL_ALREADY_EXISTS`)
- [x] 이메일 + 비밀번호로 로그인
- [x] 이메일 미존재 / 비밀번호 불일치 시 동일한 오류 반환
- [x] 비활성화 계정 로그인 차단 (403)
- [x] 리프레시 토큰으로 토큰 갱신 (Token Rotation)
- [x] 리프레시 토큰으로 로그아웃

### 토큰 관리

- [x] 리프레시 토큰 bcrypt 해시 DB 저장
- [x] 리프레시 토큰 형식: `{tokenId}:{rawToken}`
- [x] 리프레시 토큰 유효 기간: 7일
- [x] 만료된 리프레시 토큰 자동 삭제 스케줄러

### 접근 제어

- [x] `UserJwtGuard`: Bearer JWT 검증 + DB 사용자 활성 상태 확인
- [x] `@UserAuth()` 데코레이터: JWT 인증 가드 + Swagger 문서화
- [x] `@CurrentUser()` 파라미터 데코레이터: 인증된 사용자 정보 주입

### 프로필

- [x] 본인 프로필 조회 (`GET /users/me`)

### 관리자 기능

- [x] 사용자 목록 조회 (이름 검색, 활성 상태 필터, 페이지네이션)
- [x] 사용자 상세 조회
- [x] 사용자 비활성화 / 활성화

### 비밀번호 검증

- [x] 최소 8자, 최대 25자
- [x] 특수문자 1개 이상 포함 필수

## 5. 범위

### 포함

- 이메일/비밀번호 기반 회원가입 / 로그인 / 토큰 갱신 / 로그아웃 API
- JWT 기반 사용자 인증 가드
- 리프레시 토큰 DB 저장 및 해시 검증
- 만료된 리프레시 토큰 자동 정리 스케줄러
- 사용자 프로필 조회 API
- 관리자용 사용자 목록/상세 조회 + 활성 상태 관리 API

### 미포함

- OAuth / 소셜 로그인
- 비밀번호 변경 / 초기화 API
- 이메일 인증 (가입 시 이메일 확인)
- 회원 탈퇴 API
