# Rules: 서브에이전트 위임 (범용 프로세스 정책)

> 언제 위임하고, 위임받은 에이전트가 무엇을 지키는지.

## 스킬 vs 에이전트 판별 (기준)

- 판별선 = **"격리/병렬 실행에서 이득이 있느냐"** (코드를 생성하느냐가 아님).
- **에이전트**: 자기완결 + 격리/병렬 이득이 있을 때 (독립 컨텍스트·워크트리). 예) G의 API 워커(병렬), 모듈 전체를 독립 스캔하는 read-only 리뷰어.
- **스킬(책)**: 현재 컨텍스트에 밀착된 인라인 작업. 실행자가 그 자리서 책을 읽고 적용. **코드를 생성하더라도** 지금 작업 중인 대상에 종속되면 스킬이다. 예) layer 스킬들, swagger 데코레이터(방금 쓴 컨트롤러에 종속).

## 위임 시점

- **A~F**: 위임 없음. 메인 세션에서 단계별 진행 + 사람 검토 게이트(4개).
- **G (API별 구현)**: **Opus(오케스트레이터)가 Sonnet 서브에이전트에 API 단위로 위임.** 병렬.

## G 위임 규칙

- **단위 = API 1개** (= 워크트리 1개 = 브랜치 1개 = PR 1개).
- **병렬 = 디폴트 4개.** 4개 끝나면 Opus가 사람에게 "나머지 N개 진행할까요?" 확인 후 다음 배치.
- **워크트리 = Opus가 생성·네이밍·셋업(`setup-worktree.sh`)·정리** (방식 B). 실패 건 워크트리 보존.
- **Sonnet 격리 엄수**: 자기 워크트리 경로 **밖 접근 금지**. 결과는 Opus에 **반환만**.
- **공유 진행상황 문서(`G_PROGRESS.md`) 기록 = Opus 단독** (Sonnet 직접 기록 금지 → 동시쓰기 충돌 방지).
- **컨벤션 일관성**: Opus가 위임 시 **관련 스킬 + Rules를 명시 주입**(예: "application-layer 스킬 + rules/ 따르라").

## 완료 / 실패

- **완료 기준 = `lint` + `build` 통과.** (E2E는 실행 안 함 — 병렬 도커 충돌 위험, 검증은 사람이 별도.)
- Sonnet **자체 최대 5회 개선 시도** → 실패 시 실패 처리. **Opus는 자동 재투입 안 함**, 실패 기록만.
- 완료 → push + PR. 실패 → PR 없이 `G_PROGRESS.md`에 `code <경로>` + 이유.

## G_PROGRESS.md 기록 형식 (Opus가 메인 루트에 생성·기록)

G 시작 시 Opus가 메인 루트에 `G_PROGRESS.md`를 만들고 아래 형식으로 단독 기록한다(작업 종료 후 정리). 서브에이전트는 결과만 반환.

```markdown
# G 진행상황 — <모듈명>

> 부모 브랜치: <부모> | 병렬: 최대 4 | 완료 기준: lint + build (E2E 미실행)

| API        | 상태   | 워크트리 / 브랜치                      | 결과                                          |
| ---------- | ------ | -------------------------------------- | --------------------------------------------- |
| POST /...  | 진행중 | `../wt/<branch>` / `feat/<모듈>-<api>` | -                                             |
| GET /...   | 완료   | `feat/<모듈>-<api>`                    | PR #<번호>                                    |
| PATCH /... | 실패   | `code ../wt/<branch>`                  | <5회 시도 후 남은 lint/build 에러 구체적으로> |
```

- 상태: `진행중`/`완료`/`실패`. 완료→PR 링크(머지는 사람), 실패→`code <워크트리 경로>`+이유(PR 안 올림).
- 4개 완료 시 "나머지 N개 진행할까요?" 확인. 실패 워크트리는 **정리하지 않고 보존**.

## 읽기 전용 검증 에이전트

- `convention-reviewer`(read-only): 컨벤션 검증 리포트만. 격리 컨텍스트라 메인 오염 없음. (선택적 사용)
