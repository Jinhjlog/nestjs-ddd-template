# Rules: 컨벤션 원칙 (범용)

> ⚠️ **이 문서는 특정 값을 고정하지 않는다.** (ID가 UUID인지 ULID인지, status가 대문자인지 소문자인지, 역할이 무엇인지 등은 **프로젝트마다 다르다**.)
> 대신 **"어떻게 그 프로젝트의 컨벤션을 알아내고 따를 것인가"** 의 원칙을 정의한다. 실제 값은 **기존 코드 조사**로 수집한다.

## 0. 대원칙

- **컨벤션은 고정하지 말고 조사한다.** 새 코드를 짜기 전에 **기존 모듈이 실제로 어떻게 하는지** grep/조사하고 그대로 따른다. "DDD 정석"이나 "보통 이렇게 한다"로 단정하지 않는다 — 1차 근거는 **이 코드베이스의 실제 사례(파일:라인)**.
- 패턴이 둘 이상 공존하면 그 사실을 인정하고, 각 패턴이 **적합한 조건**을 근거와 함께 제시한다.
- 조사로도 확정 안 되면 **추론하지 말고** 합리적 기본값으로 진행 + `dev/context-notes.md`에 기록 → 보고 때 제시.

## 1. 조사해서 따를 항목 (값은 프로젝트마다 다름)

계약/구현 전 아래를 **조사로 수집**한다 (프로세스 D-0 체크리스트와 동일):

1. **응답 래퍼** 유무 (글로벌 인터셉터로 감싸는지 / DTO 그대로 반환)
2. **페이지네이션** 방식 (커서 vs 오프셋) + 공용 DTO/유틸 존재 여부
3. **ID 타입** (UUID/ULID/…) + 파라미터 파이프 사용 여부
4. **인증/인가** 데코레이터·가드, 역할 체계, 공개 엔드포인트 선언법
5. **enum/status 표기** (DB enum 값 대소문자 등) — _값은 조사, 검증은 VO(§2)_
6. **`@ApiProperty` 스타일** (선택 필드 표기법, enum 표기, 제약 문서화)
7. **에러 응답 형태 / errorCode** 네이밍 규칙
8. **네이밍·파일 구조** (controller/dto/transformer 위치, index 배럴, 역할별 분리 여부)
9. **`@ApiTags`** 네이밍 규칙
10. **요청 메타(IP 등) 취득 방식** — 단, **`@Ip()` 맹신 금지**: 프록시 환경에서 프록시 IP가 잡힐 수 있으니 그 프로젝트가 `x-forwarded-for` 등을 어떻게 다루는지 조사

## 2. 검증 책임 (범용 정책)

- **검증은 도메인 VO 책임. Request DTO는 타입 가드만.** (상세 `validation.md`)
- 길이/형식/enum 같은 비즈니스 규칙을 DTO의 class-validator(`@MaxLength`/`@IsEmail`/`@IsEnum`)로 막지 않는다 → VO에서. DTO엔 `@ApiProperty`로 **문서화만**.

## 3. 응답 스키마 격리 (범용 정책)

- **응답 스키마를 무지성 공유하지 마라.** 스키마 공유($ref)가 **클라이언트 코드 생성기(orval 등)나 소비자에 의도치 않은 파급**을 만들면 → **엔드포인트(역할 × 동작)별로 격리**한다 (같은 모양이어도 이름을 달리해 별도 컴포넌트로).
- 단, **잘 안 변하는 공통 코어**는 Base 타입으로 묶고 역할별로 **상속**해 격리한다(빈 상속도 가능). "공통이라 함께 바뀌어야 정상"인 것과 "무관한데 결합되는 것"을 구분.
- 중첩 객체도 전역 이름 충돌·파급을 피하도록 한다.
- ※ 단 이 격리가 **그 프로젝트에 필요한지**(codegen 사용 여부 등)는 조사로 판단.

## 4. 설계 성향 (범용 철학)

- **지역성·명시성 > 추상화·DRY.** 한 파일에서 점프 없이 이해되는 코드를 선호. 중복이 **격리**를 만든다면 가치 있는 중복으로 허용(무지성 복붙은 금지).
- "재사용하면 좋잖아요"는 근거가 아니다. **재사용이 만드는 결합**이 더 큰 비용일 수 있다.
- **단계 규율**: 현재 레이어/단계 밖 관심사를 미리 끌어오지 않는다 (`domain.md` YAGNI와 연결).
- **인라인 우선**: 고정 문자열 집합(enum 등)은 모듈 루트 상수 파일로 선점하지 말고 사용처에 인라인 — _단 이것도 기존 모듈 관례를 조사해 따른다_.
- **결정엔 근거+의도를 함께 보고** (검토 가능하게).

## 5. 컨트롤러/변환 (조사 후 따름)

- 인라인 매핑보다 **Transformer**(static) 사용이 일반적 — 기존 구조 조사해 일치시킨다.
- 인증 데코레이터가 Swagger 인증 응답(401/403)을 **자동 부착**하는지 조사 → 그렇다면 컨트롤러에서 중복 작성하지 않는다.
- 모든 하위 디렉터리 `index.ts` 배럴 등 파일 구조 관례를 따른다.
